Описание проекта интернет-магазина крупного ретейлера

1) В проекте используется одна реляционная база данных, так как данные оптимально объединить в таблицы, при этом почти все таблицы связаны между собой. 
Схема БД представлена в файле "database schema.png" и "database schema.drawio".
Медиаконтент (фотографии товаров) хранятся на внешнем сервере. В БД хранятся только ссылки на загруженные во внешний сервер фотографии.

2) Схема всего проекта представлена в файле "Схема интернет магазина.png" и "Схема интернет магазина.drawio".
Было решено разделить клиент для администратора и клиент для пользователей, чтобы не смешивать разный функционал.

3) Простой прототип доступа к товарам их бронирования создан в проекте "api_for_placing_orders".
Созданный прототип работает с помомощью простых get запросов, которые возвращают ответ в json формате.
	Get product_id=7 возвращяет product_in_warehouse с id = 7
	Get groups возвращает список существующих групп
	Get group_id=7 возвращяет список category, которые входят в группу с id = 7
	Get category_id=7 возвращяет список subcategory, которые входят в категорию с id = 7
	Get subcategory_id=7 возвращяет список product_in_warehouse, которые входят в субкатегорию с id = 7
	Get reserved_products возвращает список забронированный товаров (товары из product_in_warehouse и их статус доставки)
	Get reserved_product_id=7 бронирует товар (сохраняет его в списке забронированных товаров - reserved_product)
